# VibeVI Redesign Render Set

Purpose:
These renders are visual references for the VibeVI media-production agent and Cursor/Codex implementation pass.

Design direction:
- Make VibeVI feel like a living island-day command center, not a static travel blog.
- Use a darker cinematic app shell with coral, electric aqua, lime leaf, sand, and hibiscus accents.
- Every major page should have one strong media slot, clear decision flow, and trust-first labels.
- Generated media should describe island vibes and category context without pretending to show a specific business unless the user supplies owned/licensed media.

Rendered templates:
1. `01_homepage_redesign.png` — homepage/media/app command center.
2. `02_find_move_search_redesign.png` — search and mood-routing page.
3. `03_island_portal_redesign.png` — island hub template.
4. `04_experience_guide_redesign.png` — experience guide template.
5. `05_category_directory_redesign.png` — category/directory results template.
6. `06_business_profile_redesign.png` — real business profile template.
7. `07_map_route_board_redesign.png` — map/route board template.
8. `08_get_listed_redesign.png` — owner/get-listed landing page.
9. `00_contact_sheet.png` — overview of all renders.

Core media needs:
- Homepage hero loop: aerial water, local plate, ferry wake, harbor night.
- Four core mode cards: Beach, Boat, Bite, Night.
- Nine mood tiles: beach day, boat charter, local plate, sunset dinner, boardwalk night, culture walk, cruise day, family day, rainy day.
- Island hero sets: St. Thomas, St. Croix, St. John, Water Island.
- Experience hero sets: Adventure, Culture, Culinary/Bite, Cruise Day, Nightlife, Wellness, Stays, Local Shops.
- Category hero sets: excursions/charters, dining, stays, nightlife, wellness, provisions.
- Business profile placeholder system: cover, gallery, logo/media-rights status.
- Map system: stylized USVI map, island pins, ferry lines, category icons, route-card thumbnails.
- Trust badges: public info, unclaimed, fictional demo, confirm directly, source reviewed, no live availability.

Implementation rules:
- Do not invent real business media, reviews, contact details, live availability, booking inventory, or paid placement.
- Use owner-supplied/licensed images for real businesses.
- Use generated generic/editorial imagery for category, island, and guide pages.
- Keep all demo profiles visually labeled.
- Keep trust notes visible near booking/contact CTAs.
- Optimize final images to WebP/AVIF for web; keep PNG/SVG sources for production package.
