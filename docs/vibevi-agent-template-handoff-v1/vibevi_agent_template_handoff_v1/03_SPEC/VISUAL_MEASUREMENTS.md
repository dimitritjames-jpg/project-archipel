# Visual Measurements and Proportions

These are approximate implementation targets derived from the approved render. They are guidance, not a substitute for screenshot comparison.

## Desktop reference

- Reference viewport: 1536 × 1024 full board
- Actual desktop page reference: left ~1156px of the board
- Header: ~74px high
- Hero content area below header: ~448px
- Agent label top: ~116px
- Hero title center: ~164–205px
- Composer top: ~293px
- Composer width: ~768px
- Composer height: ~124px
- Chip row: ~434–469px
- Popular links: ~486–505px
- Explore section begins: ~522px
- Island cards: ~580–745px
- Recommendation section begins: ~778px

## Mobile reference

- Reference width: ~380px
- Header: ~82px
- Title: ~145px top, 40–44px font
- Composer: ~299px top, ~304px wide, ~115px high
- Chips: wrap into roughly 2–3 rows
- Popular links: stacked
- Ferry utility shortcut: single clean card
- Recommendation card: one large card visible

## Tolerance

Major component order and proportions should be visually close.

Do not force exact pixels when accessibility, real text, or existing media requires small changes. Document every meaningful difference.
