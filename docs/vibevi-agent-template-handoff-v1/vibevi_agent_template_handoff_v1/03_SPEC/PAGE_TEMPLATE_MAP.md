# Page Template Map

## Phase 1

| Route | Agent variant | Layout |
|---|---|---|
| `/` | `hero` | Header → agent hero → chips/popular links → islands → recommendations |
| `/search` | `search` | Header → composer → filters/results |

## Phase 2 after approval

| Page type | Agent variant | Notes |
|---|---|---|
| Island portal | `compact` | Place below island intro; keep island-specific identity |
| Experience | `compact` | Place above cards/planning prompts |
| Category directory | `compact` | Place above real filters; do not replace them |
| Business profile | `compact` or none | Keep business actions primary |
| Map | `search`/compact in list pane | Preserve Mapbox behavior |
| Ferry/cruise utility | `compact` | Utility controls remain primary |
| Guide | `compact` | Below title/intro |
| Get Listed | none in hero | Use discovery proof, not a competing visitor search bar |
| Dashboard/sign-in | out of scope | Separate owner UI decision |

## Template rule

The shared shell is consistent. Page purpose is not flattened.

A directory remains a directory.
A profile remains a profile.
A map remains a map.
The agent bar is a doorway, not the entire product.
